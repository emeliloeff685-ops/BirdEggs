# Как убрать требование скриншотов для iPad

## Проблема:
App Store Connect требует скриншоты для 13-inch iPad, хотя поддержка iPad отключена.

## Решение:

### Вариант 1: Удалить версию для iPad в App Store Connect (РЕКОМЕНДУЕТСЯ)

1. **Откройте App Store Connect:**
   - https://appstoreconnect.apple.com
   - My Apps → Bird Eggs

2. **Перейдите в Versions:**
   - App Store → Versions
   - Откройте вашу версию (или создайте новую)

3. **Проверьте раздел Platforms:**
   - Если видите "iOS (iPad)" или "iPad" - это проблема
   - Должна быть только "iOS (iPhone)"

4. **Удалите версию для iPad (если есть):**
   - Найдите раздел с iPad
   - Удалите или отключите версию для iPad
   - Оставьте только версию для iPhone

5. **Выберите правильный билд:**
   - Убедитесь, что выбран билд v1.0.18 или новее
   - Этот билд собран с `TARGETED_DEVICE_FAMILY = 1` (только iPhone)

6. **Сохраните изменения**

---

### Вариант 2: Создать новую версию с правильным билдом

1. **Создайте новую версию:**
   - App Store → Versions → "+" → New Version
   - Введите номер версии (например, 1.0)

2. **Выберите билд:**
   - Выберите билд v1.0.18 (или последний с отключенным iPad)
   - Убедитесь, что билд обработан (Ready to Submit)

3. **Проверьте Platforms:**
   - Должна быть только "iOS (iPhone)"
   - iPad не должен отображаться

4. **Заполните информацию:**
   - Загрузите скриншоты только для iPhone
   - Не загружайте скриншоты для iPad
   - Сохраните изменения

---

### Вариант 3: Подождать обработки нового билда

Если вы только что загрузили билд v1.0.18:

1. **Подождите 10-30 минут** для обработки билда
2. **Проверьте статус билда:**
   - TestFlight → iOS Builds
   - Статус должен быть "Ready to Submit"

3. **После обработки:**
   - Создайте новую версию или обновите существующую
   - Выберите обработанный билд v1.0.18
   - iPad должен автоматически исчезнуть

---

## Пошаговая инструкция для App Store Connect:

### Шаг 1: Проверьте текущую версию

1. App Store Connect → My Apps → Bird Eggs
2. App Store → Versions
3. Откройте вашу версию

### Шаг 2: Проверьте билд

1. В разделе "Build" проверьте выбранный билд
2. Убедитесь, что это билд v1.0.18 или новее
3. Если билд старый - выберите новый

### Шаг 3: Удалите iPad из версии

1. Найдите раздел "Platforms" или "Device Support"
2. Если видите iPad:
   - Удалите версию для iPad
   - Или отключите поддержку iPad
3. Оставьте только iPhone

### Шаг 4: Загрузите скриншоты только для iPhone

1. В разделе "Screenshots" загрузите скриншоты только для iPhone:
   - iPhone 6.7" (iPhone 14 Pro Max, iPhone 15 Pro Max)
   - iPhone 6.5" (iPhone 11 Pro Max, iPhone XS Max)
   - iPhone 5.5" (iPhone 8 Plus)
2. НЕ загружайте скриншоты для iPad
3. Сохраните изменения

---

## Если iPad все еще отображается:

### Проверьте настройки проекта в Xcode:

1. Откройте проект в Xcode
2. Выберите проект → Target → General
3. Проверьте "Deployment Info":
   - "Targeted Device Family" должен быть "iPhone" (не "iPhone/iPad")
   - Если видите "iPhone/iPad" - измените на "iPhone"

4. Пересоберите и загрузите новый билд

---

## Альтернативное решение: Временно загрузить скриншоты

Если ничего не помогает, можно временно загрузить скриншоты для iPad:

1. **Сделайте скриншоты:**
   - Запустите приложение на iPhone
   - Сделайте скриншоты
   - Используйте их для iPad (они будут масштабироваться)

2. **Загрузите в App Store Connect:**
   - App Store → Versions → ваша версия
   - Screenshots → iPad → 13-inch
   - Загрузите скриншоты

3. **После этого:**
   - Загрузите новый билд с отключенным iPad
   - iPad автоматически исчезнет

---

## Рекомендация:

**Лучший способ:**
1. Подождите обработки билда v1.0.18
2. Создайте новую версию в App Store Connect
3. Выберите билд v1.0.18
4. Загрузите скриншоты только для iPhone
5. Сохраните и отправьте на ревью

---

После выполнения всех шагов:

- ✅ Скриншоты для iPad не должны требоваться
- ✅ Приложение будет доступно только для iPhone
